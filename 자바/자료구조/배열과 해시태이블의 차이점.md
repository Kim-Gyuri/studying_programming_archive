# 배열과 해시태이블의 차이점

### Array
`복습 포인트`
+ 연관된 데이터를 모아서 한 번에 관리하기 위해 사용하는 데이터 타입이다.
+ 인덱스를 사용하여 해당 원소에 접근할 수 있다. 시간 복잡도 O(1)
+ 데이터를 삭제하면 빈 공간으로 남는다. 시간 복잡도 O(n)

### ArrayList
`복습 포인트`
+ 데이터를 삭제하면, 빈 공간을 메꿔주기 위해서 shift 해주어야 한다.
> 중간에 원소 삽입/삭제, 저장 공간 확장/복사  시간 복잡도 O(n)

### 배열 포인트
+ 특정 위치에 있는 원소에 대한 접근의 시간 복잡도는 O(n).
+ 배열이나 배열 리스트의 경우 연속된 메모리 공간으로 구성되어 있고, 인덱스로 데이터에 O(1)로 접근 가능했다.
+ 하지만, 특정 위치에 원소에 접근하기 위한 시간 소요가 O(n)이다. #(배열 확장을 위한 원소 삽입/삭제 과정 소요시간)


### 배열과 해시 테이블의 차이점
+ HashTable은 배열처럼 여러 개의 데이터를 저장하는 자료구조다.
+ 배열과 가장 큰 차이점은 크기가 고정되어 있지 않고 가변적이다.
+ Hash Table은 인덱스를 통한 접근이 아니라 일종의 키값을 이용해 특정 위치에 접근한다는 점이다.
